## **과제 내용**

✅**과제 1.**
```
# 카카오 쇼핑하기 웹페이지에 직접 들어가 확인해보기!

### 페이지별 구성
1. 메인 페이지
- 핵심 기능: 전체 상품 불러오기, 프로모션 및 카테고리 표시하기
- 기능 상세 설명: 쉽게 탐색할 수 있도록 추천 제품(오늘의딜, 요일별 테마딜, 전체톡딜 등), 프로모션(특별전, 쇼핑키워드, 할인제품) 및 다양한 카테고리가 포함된 레이아웃을 보여줍니다. 사용자가 특정 제품이나 매장을 찾는 데 도움이 되는 검색 기능과 키워드를 포함합니다.
- 인터페이스 요구사항: 로그인 버튼, 회전목마(캐러샐) 컴포넌트, 카테고리 메뉴, 눈에 띄는 검색 표시줄, 시각적으로 한눈에 들어오는 디자인, 다양한 카테고리에 대한 반응형 레이아웃이 필요합니다.

2. 상품 검색 결과 페이지
- 핵심 기능: 사용자가 입력한 검색 결과 표시하기
- 상세 기능 설명 : 사용자의 검색어와 일치하는 상품 목록을 보여줍니다. 상품 목록은 grid 형태로 합니다. 검색 결과를 구체화하기 위한 랭킹순, 신상품순, 낮은 가격순, 높은 가격순과 같은 정렬 옵션을 포함합니다. 각 제품의 썸네일 이미지, 제목, 가격, 리뷰개수, 만족도 및 내용을 보여줍니다. 이미지를 클릭하면 자세한 내용을 볼 수 있습니다.
- 인터페이스 요구 사항: 명확한 검색 결과 표시, 정렬 옵션, gird layout이 필요합니다.

3. 개별 상품 상세 페이지
- 핵심 기능: 특정 상품에 대한 상세 정보 제공
- 자세한 기능 설명: id값으로 상품을 조회하고, front단으로 개수 설정 옵션을 설정할 수 있게 해야합니다. 이를 통해 이미지, 설명, 가격, 배송방법, 문의 및 고객 리뷰를 포함하여 제품의 모든 관련 세부 정보를 표시합니다. 사용자가 이 페이지에서 장바구니나 위시리스트에 제품을 추가할 수 있어야합니다.
- 인터페이스 요구 사항: 상세한 상품 정보, 상품 이미지, 리뷰 및 문의창, 장바구니 또는 위시리스트에 추가하기 위한 아이콘 버튼, 구매 버튼 등이 필요합니다.

4. 장바구니 페이지
- 핵심 기능: 사용자 장바구니의 상품 관리 및 확인
- 자세한 기능 설명: 사용자의 카트에 있는 제품 목록을 수량, 이름, 주문 예상금액과 함께 표시합니다. 사용자가 수량을 업데이트하거나 항목을 제거하거나 결제를 진행할 수 있또록 합니다. 총 가격에서 즉시 할인금액 또는 톡딜 할인금액을 뺀 나머지 최종 결제 금액을 표시합니다.
- 인터페이스 요구 사항: 카트 상품 내용 요약, 편집 가능한 수량, 상품 항목 제거, 총 가격, 체크아웃이 가능한 버튼이 필요합니다.

5. 주문 목록 페이지
- 핵심 기능: 사용자의 주문 내역 및 상태 표시
- 자세한 기능 설명: 주문 날짜, 구매한 항목, 총 가격 및 현재 상태(예: 배송, 발송대기, 발송진행, 취소, 교환, 반품, 구매결정)를 포함한 세부 정보와 함께 사용자의 이전 주문 목록을 제공합니다. 사용자가 각 주문에 대한 자세한 내용을 볼 수 있게 합니다.
- 인터페이스 요구 사항: 관련 세부 정보가 포함된 주문 목록, 주문 상태 표시에 대한 5개의 버튼(전체, 발송대기, 발송진행, 취소교환반품, 구매결정)이 필요합니다.

6. 결제 페이지
- 핵심기능 : 결제정보 수집 및 처리
- 자세한 기능 설명: 사용자가 신용 카드 정보 또는 결제 세부 정보를 입력할 수 있는 안전한 결제 양식을 표시합니다. 입력된 데이터를 확인하고 안전하게 결제를 처리할 수 있게 합니다.
- 인터페이스 요구 사항: 보안상 문제가 없고, 사용자가 쉽게 결제 할 수 있도록 하는 결제 양식, 결제 정보 입력창, 잘못된 입력에 대한 오류 처리등이 필요합니다.

7. 결제 완료 페이지
- 핵심 기능: 결제 완료 확인 및 주문 내역 제공
- 상세 기능 설명 : 결제 성공을 알리는 토스트 메시지를 보여줍니다. 사용자에게 주문 번호, 구매한 항목, 총 가격, 예상 배송 날짜를 포함한 주문 세부정보를 제공해야 합니다.
- 인터페이스 요구 사항: 토스트 메시지, 보기 쉬운 주문 세부 정보 등이 필요합니다.

8. 로그인 페이지
- 핵심 기능: 로그인 요청 및 사용자 로그인 정보 저장
- 기능 상세 설명: 이메일과 비밀번호를 이용해 로그인을 진행하고, 이에 대한 상태 처리를 합니다.
- 인터페이스 요구사항: 이메일 또는 비밀번호에 들어온 값이 적합하지 않은 경우 적절한 알림을 보냅니다.

✅**과제 2.**

```
프론트 개발자가 다른 프론트 개발자와 소통 및 UI 디자이너와 소통하는데 필수적인 UI 컴포넌트의 명칭과 사용법을 익힙니다.
수업시간에 배운 컴포넌트의 명칭과 사용법 이외에 대표적인 UI 라이브러리 홈페이지를 조사해보면 수많은 컴포넌트가 어떤식으로 동작하는지 확인할 수 있습니다.
리액트 프로젝트를 생성하고, 토스트, 브래드크럼, 캐러셀, 라디오버튼, 토글버튼, 체크리스트를 UI 라이브러리가 아닌 자신만의 방식으로 스타일링하고 상태 관리를 적용해 코드를 작성하세요.
작성된 코드는 레퍼지토리에 업로드하여 멘토님에게 전달해주세요.
