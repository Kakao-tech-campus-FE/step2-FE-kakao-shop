# 페이지별 구성 설계

각 페이지마다 핵심 기능, 기능 상세 설명, 인터페이스 요구사항 작성

## 예시

```markdown
#페이지별 구성

1. 로그인 페이지

-   핵심 기능: 로그인 요청 및 사용자 로그인 정보 저장
-   기능 상세 설명: 이메일과 비밀번호를 이용해 로그인을 진행하고, 이에 대한 상태 처리를 합니다.
-   인터페이스 요구사항: 이메일 또는 비밀번호에 들어온 값이 적합하지 않은 경우 적절한 알림을 보냅니다.
    -- ...
```

## 로그인 페이지

```markdown
##로그인 페이지

-   핵심 기능: 로그인 요청 및 사용자 로그인 정보 저장
-   기능 상세 설명: 이메일과 비밀번호를 이용해 로그인을 진행하고, 이에 대한 상태 처리
-   인터페이스 요구사항: 이메일 또는 비밀번호에 들어온 값이 적합하지 않은 경우 적절한 알림
```

## 회원가입 페이지

```markdown
##회원가입 페이지

-   핵심 기능: 회원가입 요청 및 이메일 및 비밀번호 유효성 검사
-   기능 상세 설명:
    -   사용자가 입력한 이메일, 비밀번호가 유효한 형식인지 확인한다.
    -   이메일과 비밀번호가 모두 유효한 경우, 회원가입 요청을 처리한다
-   인터페이스 요구사항:
    -   유효하지 않은 입력값일 시 오류 메시지 표시를 한다.
    -   사용자에게 유효한 입력이 무엇인지 안내한다.
```

## 메인 페이지

```markdown
##메인 페이지

-   핵심 기능: 상품 목록 표시, 검색 기능, 장바구니 이동
-   기능 상세 설명:
    -   다양항 상품을 사용자에게 보여준다
    -   원하는 상품을 검색할 수 있도록 검색 창을 제공하고 검색 버튼을 누르면 해당 검색 결과 페이지로 이동한다
-   -   장바구니로 쉽게 이동할 수 있도록 장바구니 버튼을 제공한다.
-   인터페이스 요구사항:
    -   다른 페이지로 이동할 수 있는 메뉴 바나 탭을 통해 주문배송 페이지, 장바구니 페이지 등으로 이동할 수 있어야 한다
    -   검색 창은 사용자가 편리하게 접근하고 사용할 수 있도록 상단에 배치한다.
    -   장바구니 이동 버튼에 장바구니에 담긴 상품의 수를 확인할 수 있도록 한다.
```

## 검색 결과 페이지

```markdown
##검색 결과 페이지

-   핵심 기능: 검색어 관련 상품 표시
-   기능 상세 설명:
    -   사용자가 입력한 검색어와 일치하거나 유사한 상품 표시
    -   상품 이미지, 가격, 제품명 등의 핵심 정보 표시
    -   사용자가 검색 결과를 탐색할 수 있도록 스크롤 기능 추가
-   인터페이스 요구사항:
    -   사용자가 상품의 핵심 정보를 한눈에 파악할 수 있도록 이미지, 가격, 제품명 등을 표시한다.
    -   현재 사용자가 어떤 검색어를 입력했는지 상단에 표시한다.
    -   검색 결과가 많은 경우 무한 스크롤 기능을 제공하여 UX 개선
```

## 개별 상품 상세 페이지

```markdown
##개별 상품 상세 페이지

-   핵심 기능: 상품의 상세 정보 표시, 장바구니 담기, 바로 구매하기
-   기능 상세 설명:
    -   이미지, 가격, 상품명, 상품 세부 정보 등 표시하여 사용자에게 해당 상품에 대한 상세 정보 제공
    -   사용자가 해당 상품을 장바구니에 담을 수 있다. 장바구니에 추가된 상품은 별도로 저장된다.
    -   사용자가 해당 상품을 바로 구매할 수 있다. 바로 구매하기 버튼을 클릭시 결제 페이지로 이동한다.
-   인터페이스 요구사항:
    -   상품의 핵심 정보를 시각적으로 표시한다.
    -   장바구니 담기와 바로 구매하기 버튼을 추가한다.
    -   사용자가 상품을 장바구니에 담거나 바로 구매하기를 클릭한 경우, 성공적으로 수행되었음을 시각적으로 안내한다. 실패한 경우 또한 피드백을 제공한다.
```

## 장바구니 페이지

```markdown
##장바구니 페이지

-   핵심 기능: 장바구니에 담긴 상품 표시, 상품 선택 및 삭제, 결제 기능, 선택한 상품 총 가격 표시
-   기능 상세 설명:
    -   사용자가 담은 상품을 확인할 수 있도록 장바구니에 담긴 상품을 표시
    -   사용자가 삭제할 상품을 선택하고 삭제 버튼을 클릭하면 해당 상품이 장바구니에서 제거
    -   사용자가 결제 버튼을 클릭하면 선택한 상품에 대한 결제를 진행할 수 있는 결제 페이지로 이동
    -   선택한 상품의 총 가격을 표시하여 사용자가 결제 전에 가격을 확인할 수 있도록 한다.
-   인터페이스 요구사항:
    -   장바구니에 담긴 각 상품은 사용자가 장바구니의 상품을 쉽게 파악할 수 있도록 명확히 표시해야 한다
    -   사용자는 장바구니에서 원하는 상품을 선택하고 삭제할 수 있도록 체크박스를 제공한다
    -   사용자가 결제 전에 선택한 가격을 확인할 수 있도록 정확하게 표시되어야 한다
```

## 결제 페이지

```markdown
##결제 페이지

-   핵심 기능:주문 내역 확인, 배송 정보 입력, 결제 수단 선택, 결제 진행
-   기능 상세 설명:
    -   결제 전에 주문 내역을 확인할 수 있도록 표시한다.
    -   상품을 배송받을 주소 및 연락처 등의 정보를 입력한다.
    -   결제에 사용할 수단을 선택할 수 있다.
    -   배송 정보 입력과 결제 수단 선택이 완료된 뒤 사용자가 결제 버튼을 클릭하면 결제가 진행된다.
-   인터페이스 요구사항:
    -   결제가 처리되고 주문이 완료되었음을 결제 완료 페이지로 자동으로 이동하는 등의 방식으로 시각적으로 안내해야 한다.
```

## 결제 완료 페이지

```markdown
##결제 완료 페이지

-   핵심 기능: 결제 완료 안내, 주문 정보 및 결제 정보 제공
-   기능 상세 설명:
    -   주문이 정상적으로 처리되었음을 사용자게에 알리는 안내 메세지 제공
    -   사용자가 주문한 상품의 정보를 제공하여 사용자가 주문 내역을 확인할 수 있도록 한다.
    -   사용자가 어떤 결제 수단을 사용했는지와 관련된 정보를 제공한다
-   인터페이스 요구사항:
    -   메인페이지로 이동할 수 있는 버튼을 적절하게 위치시킨다.
```

## 주문배송 페이지

```markdown
##주문배송 페이지

-   핵심 기능: 주문내역 확인, 주문 반품취소 기능
-   기능 상세 설명:
    -   주문한 상품의 현재 상태와 배송 정보를 제공한다
    -   주문 취소 기능을 제공하여 사용자가 주문을 취소할 수 있도록 한다.
-   인터페이스 요구사항:
    -   주문 상태와 배송 정보는 사용자가 쉽게 확인할 수 있도록 시각적인 표시나 진행 상황 바를 활용한다
    -   주문 취소 기능을 사용자가 쉽게 찾을 수 있도록 배치한다
    -   주문 취소 시, 취소된 주문에 대한 확인 메시지를 제공한다
```

## 반품취소 페이지

```markdown
##반품취소 페이지

-   핵심 기능: 반품취소 내역 확인, 반품취소 상태 확인
-   기능 상세 설명:
    -   반품취소 신청 내역을 제공한다
    -   반품취소 상태 확인을 통해 사용자가 반품 신청이 접수되었는지, 처리 중인지, 반품이 완료되었는지 등의 정보를 확인할 수 있다.
-   인터페이스 요구사항:
    -   사용자가 어떤 상품을 반품취소 신청했는지와 해당 상품의 반품취소 상태를 쉽게 파악할 수 있어야 함
    -   반품취소 상태는 사용자가 진행 상황을 실시간으로 확인할 수 있도록 업데이트 되어야 한다
```
