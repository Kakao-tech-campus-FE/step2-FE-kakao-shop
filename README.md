# 과제 1
## 과제 1 카카오 쇼핑하기 페이지 구조 분석
### 1. 메인 페이지
- 핵심 기능
  - 각 페이지들이 연결된 메인 페이지
  - 할인중인 아이템들의 이름, 할인가격, 구매한 사람 등의 정보를 보여줌
  - 리뷰가 많은 아이템을 보여주고, 관련 리뷰를 하단에 같이 보여줌
- 기능 상세 설명
1. 상단에 있는 네비게이션 바
  - 좌측 `쇼핑하기` 버튼을 누르면 홈 화면으로 돌아감
  - 홈, 브랜드데이, 베스트, 라이브, 기획전 버튼을 누르면 해당 페이지로 이동
  - 카테고리를 누르면 상세 메뉴가 나오는 드롭다운 메뉴
  - 검색 버튼
  - 주문내역 버튼
    - 구매 or 배송중인 상품 관련 정보를 볼 수 있는 페이지로 이동
  - 장바구니 버튼
    - 장바구니에 담은 아이템을 볼 수 있는 페이지로 이동
  - 로그인 버튼
    - 카카오 로그인을 할 수 있는 페이지로 이동
    - 로그인이 된 상태이면 회원 정보를 바꿀 수 있는 상태로 변경됨
2. Carousel로 만들어진 이벤트 슬라이드
  - Slide가 자동으로 넘어가는 Carousel
  - 각 Slide를 클릭하면 해당 이벤트 관련 페이지로 이동
  - arrow control을 통해 좌, 우 slide를 넘기도록 구현
3. 하위 아이템 페이지로 넘어갈 수 있는 네비게이션
  - 해당 카테고리 영역을 클릭하면 관련 아이템을 불러오는 네비게이션 제작
4. 톡딜가 아이템 정보
  - 각 아이템은 이미지, 제품명, 할인가, 구매자수를 표시하고 / 공유하기, 찜하기 기능 추가
  - 우측에는 후기가 많은 아이템의 정보와 후기가 명시되어 있고, 페이지를 넘기는 arrow button이 있음
5. 상단 이동 버튼
  - 페이지 스크롤을 많이 내렸을 때 우측 하단에 페이지의 상단으로 이동할 수 있는 버튼 위치
6. 마감임박 톡딜 랭킹
  - 할인 마감이 얼마 남지 않은 제품 정보 명시
7. 인기브랜드 스토어
  - 제품 관련 브랜드 페이지로 넘어갈 수 있는 리스트(브랜드이미지, 브랜드명) 제작
- 인터페이스 요구사항
1. 상단에 있는 네비게이션 바
  1. 스크롤을 내려도 최상단에 위치가 고정되도록 제작
  2. 홈, 브랜드데이, 베스트, 라이브, 기획전
    1. 마우스를 올렸을 때 하단부에 bar가 생기고, 텍스트가 두꺼워짐
    2. 클릭했을 때  `/home/{page}` 로 해당 페이지 넘어가는 방식
  3. 카테고리
    1. 텍스트 색상은 파란색으로 해서 구분을 두고, 클릭했을 때 상세 카테고리를 보여주는 드롭다운 메뉴로 제작
    2. 큰 카테고리 안에 작은 카테고리로 분류
  4. 검색 버튼
    1. 검색 아이콘을 사용하고, 클릭 시 해당 레이아웃이 화면에 나오도록 제작
  5. 구매목록, 주문 조회 버튼
    1. 로그인이 되어있지 않으면, 로그인 안내 모달 창
    2. 로그인 되어있으면, 해당 페이지로 이동
  6. 장바구니 버튼
    1. 로그인이 되어있지 않으면, 로그인 안내 모달 창
    2. 로그인 되어있으면, 해당 페이지로 이동
  7. 로그인 버튼
    1. 로그인이 되어있지 않으면, 로그인 페이지로 이동
    2. 로그인 되어있으면, 카카오톡 프로필 사진, 이름이 나오도록 제작하고, 해당 부분을 클릭하면 드롭다운 메뉴가 나오도록 제작
2. Carousel로 만들어진 이벤트 슬라이드
  1. 슬라이드의 개수는 6개
  2. 현재 나오고 있는 슬라이드를 표시하기 위해 하단부에 점 인디케이터 표시
  3. 5초 간격으로 자동 슬라이드
3. 하위 아이템 페이지로 넘어갈 수 있는 네비게이션
  1. 상단부
      1. “오늘의 딜”, “만원장터Day”, “전체톡딜”로 톡딜 상품을 분류
  2. 하단부
      1. 상단부에서 선택된 카테고리에 맞는 네비게이션
          - 오늘의 딜 → 전체, 식품, 생활, 디지털, 뷰티패션, 여름리빙템
          - 만원장터 → 요일별 할인 이벤트
          - 전체톡딜 → 전체 + “최상단 네비게이션 바-카테고리”에 있는 항목
4. 톡딜가 아이템 정보
  1. 좌측 아이템 정보
      1. 각 아이템의 이미지를 클릭하면 상품 상세 페이지로 이동
      2. 아이템의 기존 가격 좌측에 할인된 가격 명시
      3. 구매자 수가 표시된 부분 우측에 공유하기와 찜하기 버튼
      4. 공유 버튼을 누르면 공유 방법을 정하는 모달 창 생성
      5. 찜하기 버튼을 누르면 색상이 변경되고, 해당 아이템의 정보가 찜한 상품으로 들어감
  2. 우측 아이템 정보 (후기 증명 BEST)
      1. BEST 아이템의 이미지와 상품명, 리뷰 정보가 나오도록 제작
      2. 한 페이지에 2개씩 12개의 페이지를 arrow 버튼으로 조절
5. 상단 이동 버튼
  1. 스크롤을 내려서 제일 위에 있는 콘텐츠를 볼 수 없게 될 위치부터 우측하단에 상단 이동 버튼이 나오도록 제작
  2. 다시 스크롤을 위로 올려서 위에 있는 콘텐츠를 볼 수 있게 될 때 상단 이동 버튼 숨김
6. 마감임박 톡딜 랭킹
  1. 이미지의 좌측 상단에는 랭킹 순위와 할인이 남은 시간 표시
  2. 제품명 위에 “무료배송”, “쿠폰할인” 등의 세부 정보를 작은 box로 넣음
  3. 기존 가격에 가운데 표시줄을 넣고, 할인된 가격을 아래에 명시
7. 인기브랜드 스토어
  - 인기브랜드의 로고이미지와 브랜드명을 수직방향으로 놓고, 브랜드 아이템들을 수평방향으로 정렬
  - 정해놓은 width를 벗어나는 브랜드명은 벗어난 텍스트를 …으로 표시
   
### 2. 카테고리별 상품 페이지
- 핵심기능
  - 카테고리별 상품 표시
- 기능 상세 설명
  1. 카테고리의 하위 목록을 보여줄 수 있는 Slide 박스 생성
  2. Slide 박스 위에 현재 카테고리를 표시하는 “Selected category” 표시
  3. 제품의 이미지, 가격, 제품명을 표시하는 제품 리스트 제작
  4. 하단부에 페이지를 넘기면서 제품을 볼 수 있도록 제작
- 인터페이스 요구사항
  1. Slide 박스
      1. 좌우로 넘기는 제스처를 통해 이후 목록을 볼 수 있도록 제작
      2. 현재 카테고리에 해당하는 텍스트를 강조 표시
  2. Selected category
      1. 현재 선택된 카테고리를 표시
      2. 모두보기 외 다른 하위 메뉴로 들어가게 되면 “Main > Sub" 형식으로 하위 메뉴까지 표시
      3. 메뉴를 클릭하면 다른 메뉴로 변경할 수 있도록 모달 창 생성
  3. 제품 정보
      1. 이미지의 우측 하단에 찜하기 버튼
      2. 가격의 경우, 제품별 가격이 다르면 (최소 가격~) 형식으로 
  4. 페이지 슬라이드
      1. 현재 페이지를 표시할 수 있도록 구분
      2. 5 페이지까지 리스트에 나오도록 하고, 이후 페이지는 5 페이지 단위로 슬라이드 버튼에 따라 조작
   
### 3. 개별 상품 상세 페이지
- 핵심 기능
    - 상품의 상세 정보를 표시하고, 해당 상품 주문
- 기능 상세 설명
    1. 스토어 정보 표시
    2. 스토어 내 검색 기능
    3. 상품 선택 및 주문 기능
    4. 상세 정보, 리뷰, 문의 표시 기능
    5. 추가 상품 표시 기능
        1. 해당 브랜드의 인기상품
        2. 해당 카테고리의 인기상품
    6. 하단부 스토어찜 기능
- 인터페이스 요구사항
  1. 스토어 정보 표시
    1. 해당 스토어의 이름을 누르면 스토어 정보 페이지로 이동
    2. “스토어 정보”를 누르면 판매자의 정보가 나오는 페이지로 이동
  2. 스토어 내 검색 기능
    1. input 영역을 focus 했을 때 색상을 다르게 구분
    2. placeholder로 “브랜드명 상품검색”을 표시
    3. 특정 단어를 검색했을 때, 해당 검색결과를 보여주는 페이지로 이동
  3. 상품 선택 및 주문 기능
    1. 스크롤을 내려도 우측에 위치가 고정될 수 있도록 제작
    2. 옵션을 선택했을 때 해당 옵션의 상품과 가격, 수량을 조절할 수 있는 버튼이 리스트로 나오도록 제작
    3. 배송비를 포함한 총 주문 금액 표시
    4. 찜하기, 장바구니에 넣기 버튼을 통해 선택한 상품을 넣을 수 있게 제작
    5. 바로 구매 버튼을 누르면 선택한 상품 결제 페이지로 이동
  4. 상세 정보, 리뷰, 문의 표시 기능
    1. 구분된 버튼 그룹으로 3가지 메뉴를 표시
    2. 각 테마에 맞는 상세 내용이 아래에 표시됨
    3. 제품 상세 정보의 경우, 일정 영역을 벗어난 설명을 펼치고 접을 수 있는 버튼 제작
    4. 리뷰의 경우, 작성된 제품 리뷰가 나오도록 제작
        1. 리뷰가 없을 땐 “작성된 리뷰가 없습니다다” 라는 표시를 넣어줌
    5. 문의의 경우, 사용자가 문의를 남길 수 있도록 문의하기 버튼을 만들고, 남긴 문의 로그가 나오도록 제작
        1. 문의가 없을 땐 “작성된 문의가 없습니다” 라는 표시를 넣어줌
  5. 추가 상품 표시 기능
    1. 해당 브랜드의 인기 상품을 4개씩 보여주고, arrow contol을 통해 좌우로 이동
    2. 해당 카테고리의 인기 상품을 4개씩 보여주고, arrow contol을 통해 좌우로 이동
   
### 4. 스토어 정보 페이지
- 핵심 기능
  - 스토어의 정보(스토어 설명, 판매중인 상품) 표시 및 카카오톡 채널 안내
- 기능 상세 설명
  1. 스토어 정보 표시
  2. 스토어 내 검색 기능
  3. 카테고리별 네비게이션(홈, 베스트, 카테고리, 기획전)
    1. 홈 화면
      1. 스토어 접근성 설정 기능(공유하기, 스토어찜, 카카오톡 소식받기)
      2. 할인 상품 정보
      3. 인기 상품 정보
      4. 신상품 정보
      5. 전체 상품 정보
          - 정렬 기준 변경 버튼
          - 상품 정보 표시 방식 변경 버튼
    2. 베스트 화면
      1. 일주일 간 판매실적에 따라 순위를 나눠 판매 상품 정렬
    3. 카테고리 화면
      1. 카테고리별 상품 정렬
      2. 정렬 기준을 바꿀 수 있는 드롭다운 메뉴
      3. 리스트 형식을 바꿀 수 있는 버튼
    4. 기획전
      1. 현재 스토어 내 진행중인 기획전 관련 정보 표시
- 인터페이스 요구사항
  1. 스토어 이름을 클릭하면 스토어 홈 페이지로 이동하고, 스토어정보를 클릭하면 판매자 정보로 이동
  2. 검색 input이 focus되면 구분될 수 있도록 표시하고, 검색 했을 때 검색 결과 페이지로 이동
  3. 현재 카테고리를 표시할 수 있도록 표시
    1. 홈 화면
      1. 스토어찜과 카톡 소식 받기 버튼을 눌렀을 때 상세 내용을 볼 수 있는 모달 창 생성
      2. 할인 상품 정보를 수평방향으로 슬라이드하여 볼 수 있도록 제작
      3. 신상품 정보는 좌우 스크롤을 통해 수평방향으로 움직이면서 상품 정보 표시
      4. 전체 상품 정보는 판매인기순을 기본 정렬 기준으로 하여 표시하고, 정렬 방식을 바꿀 수 있는 드롭다운 메뉴 제작
      5. 상품 정보 표시 방식 버튼을 눌렀을 때 상품의 표시 형식이 바로 변경될 수 있도록 제작
    2. 베스트 화면
      1. 각 상품의 순위를 이미지 좌상단에 표시
      2. 각 상품의 이미지, 제품명, 할인가격 표시
    3. 카테고리 화면
      1. 세부 카테고리로 들어가면, 카테고리 선택 box가 없어지고 현재 카테고리가 표시되도록 제작
    4. 기획전
      1. 기획전이 없을 땐 “진행중인 기획전이 없습니다” 와 같은 표시
   
### 5. 주문 조회 페이지
- 핵심 기능
  - 설정된 기간 동안의 주문, 구매, 배송중인 상품 정보 표시
- 기능 상세 설명
  1. 상품유형 결정 메뉴
  2. 주문 상태 카테고리 분류
  3. 기간, 상품 유형 결정 버튼
- 인터페이스 요구사항
  1. 상품유형 버튼을 누르면 “전체 상품유형, 이용권, 배송상품” 중 하나를 고를 수 있는 select option이 표시

## 과제 1 디렉토리 구성
### 해당 디렉토리는 React로 제작했다는 것을 가정하기 때문에 기본적인 구조를 CRA 구조에서부터 시작했다.
-  `components`의 경우, 실제로 더 많은 컴포넌트가 사용되겠지만 페이지의 메인 기능과 관련된 것 위주로 작성하였다.
-  `pages`의 경우, 카카오 쇼핑하기에 들어가서 페이지 경로를 따라 처음으로 route되는 부분 위주로 작성하였다.
`ex> stoe.kakao.com/home -> home page가 필요`
-  `hooks & config`에는 해당 내용이 작성된 js 파일로 관리를 하도록 했다.
- 기존 CRA 구조 생성 시 만들어진 CSS 파일은 `styles`에 넣었다. 하지만 강의 때도 말씀하셨듯이 CSS 파일을 직접 다루는 일은 적기 때문에 styles 폴더는 생략해도 될 것 같다고 생각한다.
-  `api`의 경우, 메인 기능인 로그인과 결제 관련 API만 명시하였고, 추가적인 기능으로는 카카오의 주요 기능을 가져오는 카카오 관련 API가 있을 것 같다.
-  `assets`에는 보통 이미지나 폰트와 같은 파일이 들어가기 때문에, 해당 부분의 내용만 넣었다.
-  `utils`에는 정규식이나 공통으로 사용할 수 있는 함수를 따로 빼서 사용한다. 공통으로 사용할 수 있는 함수에는 계산함수나 시간을 표시하는 countdown 정도가 들어갈 것 같아서 표시하였다.
